# QuuBee上のOPNA(YM2608)直接I/O音源プログラミング

市販/フリーDOSドライバ(PMD/MXDRV等)には依存せず、OPNAレジスタへの直接I/O
ポート書き込みで自作プレイヤーを作る場合の要点。ポート配置は
`core/np2kai/cbus/board86.c`(QuuBeeのサブモジュール、AZO234/NP2kai)で確認済み。
QuuBeeのハードウェアI/Oエミュレーションは(DOSコールと違い)HLE対象ではなく
実機同様に動作するので、以下は実機PC-98と同じ考え方でよい。

## ポート配置

オンボード音源として`0x188`固定ベースで登録される:
- `0x188` = レジスタセットA(ch1-3) アドレス書き込み/ステータス読み出し
- `0x18A` = レジスタセットA データ書き込み
- `0x18C` = レジスタセットB(ch4-6拡張) アドレス書き込み
- `0x18E` = レジスタセットB データ書き込み

Watcom Cでは`<conio.h>`の`outp(port, data)`/`inp(port)`が使える。

## 最小限の発音手順

1. `outp(addr_port, レジスタ番号); outp(data_port, 値);` でレジスタ書き込み
   (アドレス書き込み後、実機同様に短いウェイトを挟むのが安全)
2. アルゴリズム/フィードバック(レジスタ`0xB0`+チャンネル): 例えば`0x07`
   (アルゴリズム7=4オペレータ全てキャリアの加算合成)にすると、使わない
   スロットをTL=127(最大減衰=無音)にしておくだけで、スロット4だけを
   鳴らす最小構成が組める。
3. スロット4(キャリア)のTL(音量,`0x40`系)/AR・RS(`0x50`系)/DR(`0x60`系)/
   SR(`0x70`系)/SL・RR(`0x80`系)/SSG-EG(`0x90`系、使わないなら0)を設定。
4. F-Number/Block(`0xA0`/`0xA4`系)で音高を設定。
5. キーオン/オフはレジスタ`0x28`(値=スロットビット|チャンネルコード)。
   ch0-2は`0x28|チャンネル番号`、ch3-5(セットB)は`0x28|(0x04|チャンネル番号)`。

## 重要な制約: キャリアのMULを変えるとピッチがずれる

FMオペレータの「キャリア」として使うスロットのMUL(周波数逓倍, DT/MUL
レジスタの下位ニブル)を変えると、**そのスロット自体の発音ピッチがずれる**。
単一キャリアのみを使う最小構成では、MULで音色を変えるという発想は使えない。
**音色の違いはAR/DR/SR/SL/RR(エンベロープ)とTL(音量)だけで作ること。**
キャリアのDT/MULは常に固定(`0x01`=DT0,MUL1等)にする。

## Fnum/Block(音高)の計算式

```
Fnum = freq(Hz) * 144 * 2^(20-Block) / MasterClock
```
MasterClockはPC-98オンボードOPNAの標準値`7987200`Hz。Block(0-7)は
Fnumが0-2047の範囲に収まるよう選ぶ(オクターブごとにBlockを+-1調整)。

音名→Fnum/Blockのテーブルは`scripts/gen_notes.py`で自動生成できる
(C2-C6の範囲、`NOTE_C4`等のマクロ名で参照可能なヘッダ/実装ファイルを出力)。

## テンポ/タイミングの正確さについて

ソフトウェアビジーウェイト(`for(i=0;i<N;i++);`のようなループ)でBGMのテンポを
作ると、**QuuBeeの実際のCPUエミュレーション速度に依存するため、机上の計算値は
全く当てにならない**。「132BPM相当を狙って計算したwait値」が、実際にQuuBee上で
聴くと「その半分以下でようやく合う」ということが普通に起こる。

**対策:** 最初から「実機(QuuBee)で聴いて合わせる」前提で、控えめな値から始めて
だんだん速める方向で調整する方が手数が少ない。BPMを計算で追い込もうとしないこと。

一方、**「1秒おき」「5秒後に自動遷移」のような正確な実時間**が必要な処理では、
ビジーウェイトではなく**DOS実時刻 AH=2Ch(get time)**を使うべき。`CH:CL:DH:DL`=
時:分:秒:1/100秒が返るので、センチ秒(1/100秒)単位の値にまとめて経過時間を
比較する。日付をまたぐケースも一応考慮しておくとよい(`elapsed = now>=start ?
now-start : (86400*100 - start) + now`)。
